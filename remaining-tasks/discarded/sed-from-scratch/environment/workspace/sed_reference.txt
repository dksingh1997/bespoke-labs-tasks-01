NAME
       sed - stream editor for filtering and transforming text

SYNOPSIS
       sed [OPTION]... 'SCRIPT' [INPUT-FILE]...
       sed [OPTION]... -e 'SCRIPT' -e 'SCRIPT' [INPUT-FILE]...
       sed [OPTION]... -f SCRIPT-FILE [INPUT-FILE]...

DESCRIPTION
       Sed reads input line-by-line into a buffer called the "pattern
       space," applies the script commands in sequence, then writes the
       pattern space to standard output. A second buffer called the
       "hold space" is available for temporary storage and starts empty.

       If no input files are given, sed reads from standard input. When
       multiple files are named, they are read as a single concatenated
       stream (line numbering does not reset between files).

OPTIONS
       -n     Suppress automatic printing of the pattern space at the
              end of each cycle. Only explicit p or P commands produce
              output.

       -e SCRIPT
              Add SCRIPT to the list of commands to execute. Multiple
              -e options are applied in order.

       -f SCRIPT-FILE
              Read commands from SCRIPT-FILE instead of the command
              line.

       -i[SUFFIX]
              Edit files in place. If SUFFIX is provided, a backup of
              each original file is saved with that suffix appended.
              Without SUFFIX, no backup is made.

ADDRESSES
       Most commands accept zero, one, or two addresses that restrict
       which input lines the command acts on.

       NUMBER
              Match only line NUMBER (1-based).

       $      Match the last line of input.

       /REGEXP/
              Match any line whose contents match the regular
              expression REGEXP.

       ADDR1,ADDR2
              Match a range starting at the line matching ADDR1 and
              ending at the line matching ADDR2 (inclusive). Ranges may
              use any combination of line numbers, $ and /REGEXP/.

       ADDR!  Negation: the command is applied to every line that does
              NOT match ADDR.

COMMANDS
   Commonly used
       s/REGEXP/REPLACEMENT/[FLAGS]
              Substitute: replace text matching REGEXP with REPLACEMENT.

              In REPLACEMENT, & stands for the entire matched text;
              \1 through \9 refer to captured groups in REGEXP; \n
              inserts a literal newline.

              Any character may be used as the delimiter in place of /.

              FLAGS may include:
                g   Replace all non-overlapping matches, not just the
                    first.
                N   (a digit) Replace only the Nth match.
                p   Print the pattern space if a substitution was made.
                w FILE  Write the pattern space to FILE if a
                    substitution was made.
                I   Case-insensitive matching.

       d      Delete the pattern space and begin the next cycle
              immediately (no output for this line).

       p      Print the pattern space to standard output.

       q      Print the pattern space (unless -n), then quit.

       y/SRC/DST/
              Transliterate: replace every character in SRC with the
              corresponding character in DST. SRC and DST must be the
              same length.

   Line information
       =      Print the current input line number (followed by a
              newline) to standard output.

   Text insertion
       a\TEXT Append TEXT after each selected line's output. A literal
              newline in TEXT can be included via a backslash-newline
              continuation.

       i\TEXT Insert TEXT before each selected line's output.

       c\TEXT Replace the selected line(s) with TEXT. When applied to
              a range, the text is written once at the end of the
              range.

   File I/O
       r FILE After printing the pattern space, append the contents
              of FILE to the output.

       w FILE Write the pattern space to FILE. The file is created or
              truncated on the first w to that filename, and subsequent
              writes append.

   Hold space
       h      Copy the pattern space into the hold space (overwrite).
       H      Append a newline and the pattern space to the hold space.
       g      Copy the hold space into the pattern space (overwrite).
       G      Append a newline and the hold space to the pattern space.
       x      Exchange the contents of the pattern and hold spaces.

   Multi-line
       N      Append a newline and the next input line to the pattern
              space. If there is no more input, the default behavior
              is to write the pattern space and exit.

       P      Print up to the first embedded newline of the pattern
              space.

       D      Delete up to the first embedded newline of the pattern
              space, then restart the script from the beginning with
              the remainder (without reading a new line). If the
              pattern space becomes empty, a new line is read.

   Flow control
       :LABEL Define a label for branching commands.

       b LABEL
              Branch (jump) to LABEL. If LABEL is omitted, branch to
              the end of the script (effectively skipping remaining
              commands for this cycle).

       t LABEL
              Branch to LABEL only if a successful substitution has
              been made since the last input line was read or the last
              t was taken.

   Grouping
       { CMD1; CMD2; ... }
              Group commands so they share the same address or address
              range. Semicolons separate commands within a group.
              Braces may also span multiple lines.

REGULAR EXPRESSIONS
       Sed uses POSIX Basic Regular Expressions (BRE):

       .      Any single character.
       *      Zero or more of the preceding element.
       ^      Anchors to the beginning of the pattern space.
       $      Anchors to the end of the pattern space (or line).
       [LIST] Character class: matches any character in LIST.
              [^LIST] matches any character NOT in LIST.
       \(RE\) Grouping (capturing). Back-references \1 .. \9 refer to
              the text matched by the Nth group.
       \{N\}  Exactly N repetitions of the preceding element.
       \{N,\} N or more repetitions.
       \{N,M\} Between N and M repetitions (inclusive).
       \+     One or more of the preceding element (GNU extension).
       \?     Zero or one of the preceding element (GNU extension).

       Inside character classes, ] must appear first (or after ^) to
       be treated as literal. - is literal at the start or end.
       POSIX named classes like [:alpha:], [:digit:], [:space:] are
       supported inside bracket expressions.

EXAMPLES
       Print only lines containing "error":
              sed -n '/error/p' logfile

       Delete blank lines:
              sed '/^$/d' input.txt

       Replace "foo" with "bar" everywhere:
              sed 's/foo/bar/g' input.txt

       Number all non-empty lines:
              sed '/./=' input.txt | sed 'N;s/\n/\t/'

       Reverse line order (uses hold space):
              sed '1!G;h;$!d' input.txt

       Join every pair of lines:
              sed 'N;s/\n/ /' input.txt

       Collapse runs of duplicate characters:
              sed ':loop; s/aa/a/; t loop'

       In-place edit with backup:
              sed -i.bak 's/old/new/g' file.txt

EMPTY REGEX
       When a regex is empty (e.g. // in an address, or s///), sed
       recalls the last regular expression that was successfully used
       in any command. This applies to both address matching and the
       s command: the last *executed* regex is reused, not the last
       one compiled. For example:

              s/foo/bar/;/baz/d;s//qux/

       The final s// reuses /baz/ (the last regex executed, by the
       address), not /foo/.

NOTES
       The pattern space and hold space grow dynamically; there is no
       fixed size limit. A well-behaved implementation handles input of
       arbitrary length without truncation.

       When -n is active and no print command executes, the line is
       silently consumed. The q command respects -n: with -n, q exits
       without printing.

       Multiple -e options, -f script files, and semicolons within
       expressions all contribute to a single script that is executed
       in order for each input line.

       Sed processes 8-bit data. Characters outside the ASCII printable
       range should pass through unmodified. Binary data (including NUL
       bytes) may appear in input.
